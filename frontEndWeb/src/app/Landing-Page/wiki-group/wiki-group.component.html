Arrienda tu Finca - Documentación

1. Introducción

El proyecto "Arrienda tu Finca" busca digitalizar y simplificar el proceso de arrendamiento de fincas, permitiendo la interacción entre dos perfiles principales: arrendadores y arrendatarios. La plataforma permite publicar fincas, gestionar solicitudes de arrendamiento, realizar seguimientos de pagos y generar contratos de arrendamiento.

Esta documentación describe la arquitectura del sistema, la implementación de la autenticación JWT, las pruebas de interfaz y las pruebas de carga realizadas para garantizar la seguridad y el rendimiento de la aplicación.

El proyecto está desarrollado bajo una arquitectura de tres capas:

  • Frontend: construido en Angular para ofrecer una experiencia de usuario dinámica y responsiva.
  • Backend: desarrollado en Java utilizando Spring Boot, con conexión a una base de datos relacional.
  • Base de Datos: MySQL como sistema de gestión de bases de datos para almacenar y gestionar la información del sistema.

2. Funcionalidades

La plataforma "Arrienda tu Finca" ofrece una variedad de funcionalidades para facilitar la interacción entre arrendadores y arrendatarios, y agilizar el proceso de arrendamiento de fincas. Estas funcionalidades se dividen en dos categorías principales: funcionalidades para arrendadores y funcionalidades para arrendatarios.

  • Gestión de Perfil:
    • Crear y actualizar su perfil con información personal y de contacto.
    • Subir una foto de perfil para personalizar su cuenta.
    • Gestionar la información de su cuenta, como la contraseña y la dirección de correo electrónico.

      • Nombre de la finca
      • Ubicación (dirección, coordenadas geográficas)

      • Tipo de finca (ganadera, agrícola, recreativa)
      • Características (casa principal, instalaciones, servicios)
      • Precio de arrendamiento (por día, semana, mes)
      • Fotos de la finca
      • Disponibilidad (calendario de fechas disponibles)
    • Editar o eliminar anuncios de fincas existentes.
  • Gestión de Solicitudes:
    • Recibir notificaciones de nuevas solicitudes de arrendamiento.
    • Revisar las solicitudes de arrendamiento, incluyendo la información del arrendatario y las fechas solicitadas.
    • Aceptar o rechazar las solicitudes de arrendamiento.
    • Comunicarse con los arrendatarios a través de un sistema de mensajería interno para aclarar dudas o negociar términos.
  • Gestión de Contratos:
    • Generar contratos de arrendamiento automáticamente cuando se acepta una solicitud.
    • Acceder a los contratos generados en formato digital.
    • Descargar o imprimir los contratos para su firma.
    • Llevar un registro de los contratos de arrendamiento, incluyendo el estado del contrato (pendiente, activo, finalizado).
  • Calificación de Arrendatarios:
    • Calificar a los arrendatarios después de finalizar un contrato de arrendamiento.
    • Proporcionar una calificación y una reseña sobre la experiencia con el arrendatario.
    • Ayudar a otros arrendadores a tomar decisiones informadas sobre los posibles arrendatarios.
  • Historial de Arrendamientos:
    • Acceder a un historial de todos los arrendamientos de sus fincas.
    • Ver la información de los arrendatarios, las fechas de arrendamiento y el estado del contrato.
    • Consultar los pagos recibidos por los arrendamientos.

3. Implementación de JWT

La seguridad es un aspecto crucial en "Arrienda tu Finca". Para garantizar la autenticación robusta de usuarios y la protección de la información sensible, se ha implementado la autenticación JWT (JSON Web Tokens). JWT es un estándar abierto (RFC 7519) que define una forma compacta y autocontenida para transmitir información de forma segura entre partes como un objeto JSON.

En lugar de utilizar sesiones tradicionales del lado del servidor, que pueden ser vulnerables a ataques y presentan desafíos de escalabilidad, JWT permite una autenticación sin estado. Esto significa que el servidor no necesita almacenar información de sesión, lo que facilita la escalabilidad y el mantenimiento de la aplicación.

¿Cómo funciona JWT en "Arrienda tu Finca"?

  1. Inicio de sesión: Cuando un usuario inicia sesión con sus credenciales válidas (correo electrónico y contraseña), el backend genera un token JWT. Este token contiene información sobre el usuario, como su ID, rol (arrendador o arrendatario) y cualquier otra información relevante para la autorización.
  2. Envío del token: El token JWT se envía al cliente en la respuesta de la solicitud de inicio de sesión, generalmente dentro del cuerpo de la respuesta HTTP o en una cabecera específica.
  3. Almacenamiento del token: El cliente (el navegador web) almacena el token JWT de forma segura, generalmente en el almacenamiento local (localStorage) o en una cookie HttpOnly. Esto permite que el token se mantenga de forma persistente entre diferentes sesiones del usuario.
  4. Inclusión del token en las solicitudes: En cada solicitud posterior a recursos protegidos del backend, el cliente incluye el token JWT en la cabecera Authorization de la solicitud HTTP. Generalmente se utiliza el formato "Bearer [token JWT]", donde "Bearer" es el esquema de autenticación.
  5. Validación del token: El backend intercepta la solicitud, extrae el token JWT de la cabecera Authorization y lo valida. La validación incluye:
    • Verificar la firma del token para asegurarse de que no ha sido modificado.
    • Verificar que el token no ha expirado.
    • Extraer la información del usuario del token, como el ID y el rol.
    Si el token es válido, se concede el acceso al recurso solicitado. En caso contrario, se devuelve un error de autenticación (generalmente un código de estado HTTP 401 No autorizado).

Este proceso garantiza que solo los usuarios autenticados con tokens JWT válidos puedan acceder a las funcionalidades protegidas de la aplicación. Además, al ser sin estado, la aplicación puede escalar fácilmente sin necesidad de gestionar sesiones en el servidor.

Componentes de la Implementación JWT

Para la implementación de JWT en "Arrienda tu Finca", se han utilizado las siguientes clases y componentes:

  • SecurityConfig:
    • Clase principal que configura la seguridad de la aplicación utilizando Spring Security.
    • Define las rutas que requieren autenticación (rutas protegidas) y las que son públicas (rutas a las que se puede acceder sin autenticación).
    • Configura el filtro JWTAuthenticationFilter para la autenticación JWT, que se encarga de interceptar las solicitudes y validar los tokens.
    • Define el JwtAuthEntryPoint para manejar los errores de autenticación, que se encarga de devolver una respuesta de error al cliente cuando la autenticación falla.
    • Configura el PasswordEncoder para el cifrado de contraseñas, que se utiliza para cifrar las contraseñas de los usuarios antes de almacenarlas en la base de datos.
    • Deshabilita la protección CSRF (Cross-Site Request Forgery) ya que JWT, al no usar cookies de sesión, no es vulnerable a este tipo de ataque.
  • JWTAuthenticationFilter:
    • Filtro que se ejecuta para cada solicitud HTTP.
    • Intercepta las solicitudes y extrae el token JWT de la cabecera Authorization.
    • Valida el token JWT utilizando JWTTokenService.
    • Si el token es válido, obtiene los detalles del usuario (ID, roles, etc.) del token.
    • Crea un objeto UsernamePasswordAuthenticationToken con los detalles del usuario y lo autentica en el contexto de Spring Security, permitiendo el acceso a los recursos protegidos.
  • JwtAuthEntryPoint:
    • Clase que implementa AuthenticationEntryPoint de Spring Security.
    • Se encarga de manejar los errores de autenticación, como cuando se intenta acceder a un recurso protegido sin un token JWT válido o con un token expirado.
    • Personaliza la respuesta de error que se envía al cliente, generalmente un código de estado HTTP 401 (No autorizado) y un mensaje de error descriptivo.
  • JWTTokenService:
    • Clase responsable de generar y validar los tokens JWT.
    • Utiliza la librería jjwt para firmar y verificar los tokens, garantizando su integridad y autenticidad.
    • Define la clave secreta utilizada para firmar los tokens, que debe mantenerse segura y confidencial.
    • Define el tiempo de expiración de los tokens, después del cual se consideran inválidos.
    • Proporciona métodos para extraer la información del usuario del token JWT, como el ID, el rol y otras claims (reclamaciones) personalizadas.

JWTAuthenticationFilter

package com.example.ProyectoWeb.config;
                
                import java.io.IOException;
                
                import org.springframework.beans.factory.annotation.Autowired;
                import org.springframework.security.authentication.UsernamePasswordAuthenticationToken;   
                
                import org.springframework.security.core.context.SecurityContextHolder;   
                
                import org.springframework.security.core.userdetails.UserDetails;
                import org.springframework.security.web.authentication.WebAuthenticationDetailsSource;
                import org.springframework.web.filter.OncePerRequestFilter;
                
                import jakarta.servlet.FilterChain;
                import jakarta.servlet.ServletException;
                import jakarta.servlet.http.HttpServletRequest;
                import jakarta.servlet.http.HttpServletResponse;
                
                public class   
                    JWTAuthenticationFilter extends OncePerRequestFilter   
                    {
                
                    @Autowired
                    private JWTTokenService jwtTokenService;
                
                    @Autowired
                    private CustomUserDetailsService customUserDetailsService;
                
                    private static final String HEADER = "Authorization";
                    private static final String PREFIX = "Bearer ";
                
                    @Override
                    protected void doFilterInternal(HttpServletRequest request, HttpServletResponse response, FilterChain filterChain) throws ServletException,   
                    IOException 
                    {
                        String token = getJWT(request);
                        if   
                (token != null && jwtTokenService.validarToken(token))
                        {
                            String id = jwtTokenService.getUserId(token);
                            UserDetails userDetails = customUserDetailsService.loadUserByUsername(id);
                            UsernamePasswordAuthenticationToken authToken = new UsernamePasswordAuthenticationToken(userDetails,null, userDetails.getAuthorities());
                            authToken.setDetails(new WebAuthenticationDetailsSource().buildDetails(request));
                            SecurityContextHolder.getContext().setAuthentication(authToken);
                        }
                        filterChain.doFilter(request, response);   
                
                    }
                    
                    private String getJWT(HttpServletRequest request) {
                        String authHeader = request.getHeader(HEADER);   
                
                        System.out.println("\nHeader: " + authHeader);
                        if(authHeader != null && authHeader.startsWith(PREFIX))
                        {
                            return authHeader.replace(PREFIX, "");
                        }
                        return null; 
                    }
                    
                }

JWTTokenService

package com.example.ProyectoWeb.config;
                
                import java.security.Key;
                import java.util.ArrayList;
                import java.util.Collection;
                import java.util.Date;
                import java.util.stream.Collectors;   
                
                
                import org.springframework.security.core.GrantedAuthority;
                import org.springframework.stereotype.Component;
                
                import com.fasterxml.jackson.databind.ObjectMapper;
                
                import io.jsonwebtoken.Claims;
                import io.jsonwebtoken.Jwts;
                import io.jsonwebtoken.SignatureAlgorithm;   
                
                import io.jsonwebtoken.security.Keys;   
                
                
                @Component
                public class JWTTokenService {
                
                    private long jwtExpiration = 99999999;
                    private Key jwtKey = Keys.secretKeyFor(SignatureAlgorithm.HS512);; 
                
                    public String generarToken(int id, String userType) {
                
                        ObjectMapper objectMapper = new ObjectMapper();
                        String userID = "";
                        try {
                            userID = objectMapper.writeValueAsString(id); 
                        } catch (Exception e) {
                            e.printStackTrace();
                        }
                
                        Date now = new Date();
                        Date expiryDate = new Date(now.getTime() + jwtExpiration); 
                
                        Collection<? extends GrantedAuthority> authorities = new ArrayList<>(); 
                
                        return Jwts.builder()
                            .setSubject(userID) 
                            .setIssuedAt(now) 
                            .setExpiration(expiryDate)
                            .claim("userType", userType) 
                            .claim("authorities", authorities.stream()
                                .map(GrantedAuthority::getAuthority)
                                .collect(Collectors.toList())) 
                            .signWith(jwtKey, SignatureAlgorithm.HS512) 
                            .compact(); 
                    }
                
                    public String getUserId(String jwtToken){
                        System.out.println("\nsubject: "+decodificarToken(jwtToken).getSubject());
                        return decodificarToken(jwtToken).getSubject();
                    }
                    public Date getFechaExpiracion(String jwtToken){
                        return decodificarToken(jwtToken).getExpiration();
                    }
                    
                    public String getUserType(String jwtToken) {
                        return decodificarToken(jwtToken).get("userType", String.class); 
                    }
                
                    public Claims decodificarToken(String jwtToken) {
                
                        return Jwts.parserBuilder()
                            .setSigningKey(jwtKey)
                            .build()
                            .parseClaimsJws(jwtToken)
                            .getBody();
                    }
                    public boolean validarToken(String token)
                    {
                        try
                        {
                            decodificarToken(token);
                            return true;
                        }catch(Exception e)
                        {
                            return false;
                        }
                    }
                }

Beneficios de la Implementación JWT

  • Seguridad mejorada: JWT proporciona un mecanismo de autenticación robusto y seguro, que protege la aplicación contra ataques comunes como la falsificación de solicitudes entre sitios (CSRF) y la reutilización de tokens.
  • Escalabilidad: La autenticación sin estado facilita la escalabilidad de la aplicación, ya que el servidor no necesita mantener información de sesión para cada usuario. Esto permite que la aplicación maneje un gran número de usuarios concurrentes sin comprometer el rendimiento.
  • Eficiencia: Los tokens JWT son compactos y se transmiten fácilmente en las solicitudes HTTP, lo que reduce la sobrecarga de comunicación entre el cliente y el servidor.
  • Flexibilidad: JWT se puede utilizar para la autenticación en diferentes tipos de aplicaciones y plataformas, incluyendo aplicaciones web, móviles y APIs.
  • Información autocontenida: El token JWT contiene la información necesaria para la autenticación y autorización, lo que reduce la necesidad de realizar consultas a la base de datos en cada solicitud.
  • Fácil implementación: Existen librerías y herramientas disponibles para facilitar la implementación de JWT en diferentes lenguajes de programación y frameworks.

La implementación de JWT en "Arrienda tu Finca" contribuye significativamente a la seguridad y la eficiencia de la plataforma, garantizando una experiencia de usuario confiable y protegida.

4. Pruebas de Interfaz

Para asegurar la calidad y la funcionalidad del frontend de "Arrienda tu Finca", se ha llevado a cabo un conjunto exhaustivo de pruebas de interfaz utilizando Cypress. Cypress es una herramienta de pruebas de extremo a extremo (E2E) de código abierto que permite simular la interacción del usuario con la aplicación en un navegador real.

Estas pruebas se centran en validar diferentes aspectos de la interfaz de usuario, incluyendo:

  • Renderización de Componentes:
    • Verificar que todos los componentes de la interfaz de usuario se renderizan correctamente en diferentes navegadores y dispositivos.
    • Comprobar que los elementos de la interfaz, como botones, campos de entrada, imágenes y tablas, se muestran de forma correcta y en la posición esperada.
    • Validar que los estilos CSS se aplican correctamente y que la interfaz tiene la apariencia deseada.
  • Interacción del Usuario:
    • Simular las acciones del usuario, como clics, ingreso de datos en formularios, selección de elementos, desplazamiento de la página, etc.
    • Verificar que la aplicación responde correctamente a las interacciones del usuario, realizando las acciones esperadas, como mostrar mensajes de error, actualizar la información en la pantalla o redirigir a otras páginas.
    • Validar la lógica de la interfaz de usuario, como la validación de formularios, el manejo de eventos y la actualización del estado de la aplicación.
  • Flujo de Navegación:
    • Comprobar el flujo de navegación entre las diferentes vistas de la aplicación, simulando el recorrido del usuario a través de las diferentes páginas.
    • Verificar que las redirecciones entre páginas funcionan correctamente, tanto en la navegación normal como en situaciones especiales, como después de iniciar sesión o enviar un formulario.
    • Asegurar que el usuario puede acceder a todas las secciones de la aplicación de forma fluida y sin errores.
  • Integración con el Backend:
    • Validar la comunicación entre el frontend y el backend, simulando las solicitudes HTTP que se realizan al servidor.
    • Comprobar que la aplicación envía las solicitudes correctas al backend, con los datos adecuados y en el formato esperado.
    • Verificar que la aplicación maneja las respuestas del backend de forma correcta, mostrando la información recibida del servidor o mostrando mensajes de error en caso de fallos.
    • Validar la autenticación JWT, comprobando que el token se incluye en las solicitudes a recursos protegidos y que la aplicación maneja correctamente las respuestas de autenticación del backend.
  • Manejo de Errores:
    • Simular situaciones de error, como la introducción de datos inválidos en formularios o la falta de conexión a internet.
    • Verificar que la aplicación muestra mensajes de error claros y útiles al usuario en caso de errores.
    • Comprobar que la aplicación se recupera de los errores de forma correcta y permite al usuario continuar utilizando la aplicación.

Ventajas de Cypress

Cypress ofrece varias ventajas para las pruebas de interfaz:

  • Facilidad de uso: Cypress tiene una sintaxis sencilla y fácil de aprender, lo que facilita la escritura de pruebas.
  • Ejecución rápida: Cypress ejecuta las pruebas de forma rápida y eficiente, lo que permite obtener resultados rápidos.
  • Depuración sencilla: Cypress proporciona herramientas de depuración que facilitan la identificación y corrección de errores en las pruebas.
  • Integración con el navegador: Cypress se ejecuta directamente en el navegador, lo que permite una simulación precisa de la interacción del usuario.
  • Capturas de pantalla y videos: Cypress puede tomar capturas de pantalla y videos de las pruebas, lo que facilita la visualización del comportamiento de la aplicación.

5. Pruebas de Carga

Para garantizar que "Arrienda tu Finca" pueda manejar un gran número de usuarios y transacciones de forma eficiente, se realizaron pruebas de carga utilizando Apache JMeter. JMeter es una herramienta de código abierto que permite simular la carga de usuarios concurrentes en una aplicación web.

Las pruebas de carga se centraron en evaluar el rendimiento del sistema bajo diferentes niveles de estrés, simulando escenarios realistas de uso de la aplicación. Esto permitió:

  • Identificar cuellos de botella: Detectar los puntos débiles del sistema que limitan su rendimiento, como tiempos de respuesta lentos en determinadas peticiones, errores de conexión a la base de datos o saturación de recursos del servidor (CPU, memoria, ancho de banda).
  • Medir tiempos de respuesta: Obtener métricas precisas sobre los tiempos de respuesta de la aplicación bajo diferentes niveles de carga. Esto permite determinar la capacidad del sistema para manejar un gran número de usuarios concurrentes sin degradar la experiencia de usuario.
  • Evaluar la escalabilidad: Determinar cómo se comporta la aplicación al aumentar el número de usuarios o el volumen de datos. Esto permite identificar las limitaciones de la arquitectura y planificar la escalabilidad del sistema para futuras demandas.
  • Analizar el consumo de recursos: Monitorizar el consumo de recursos del servidor (CPU, memoria, ancho de banda, etc.) bajo diferentes niveles de carga. Esto permite optimizar la infraestructura y la configuración del sistema para un rendimiento óptimo.
  • Validar la estabilidad del sistema: Comprobar que la aplicación se mantiene estable bajo una carga sostenida de usuarios, sin presentar errores o fallos inesperados.

Escenarios de Prueba

Se configuraron diferentes escenarios de prueba en JMeter para simular situaciones realistas de uso de la aplicación, incluyendo:

  • Inicio de sesión concurrente: Simular un gran número de usuarios iniciando sesión en la aplicación simultáneamente.
  • Búsqueda de fincas: Simular múltiples usuarios buscando fincas con diferentes criterios de búsqueda.
  • Solicitud de arrendamiento: Simular múltiples usuarios enviando solicitudes de arrendamiento a diferentes fincas.
  • Gestión de solicitudes: Simular arrendadores gestionando múltiples solicitudes de arrendamiento (aceptando o rechazando).
  • Generación de contratos: Simular la generación de múltiples contratos de arrendamiento.

Configuración de JMeter

Para cada escenario de prueba, se configuró JMeter con los siguientes elementos:

  • Plan de Pruebas: Define la estructura general de la prueba, incluyendo los elementos de configuración, los grupos de hilos y los listeners.
  • Grupo de Hilos: Simula un grupo de usuarios virtuales que acceden a la aplicación. Se configura el número de hilos (usuarios), el tiempo de ramp-up (tiempo que tarda en iniciar todos los hilos) y el número de iteraciones (veces que cada hilo ejecuta la prueba).
  • Peticiones HTTP: Simulan las solicitudes HTTP que los usuarios realizan a la aplicación. Se configuran las peticiones a diferentes endpoints del backend, incluyendo la autenticación, la búsqueda de fincas, la gestión de solicitudes y la generación de contratos.
  • Gestor de Cabeceras HTTP: Permite configurar las cabeceras HTTP de las peticiones, incluyendo la cabecera Authorization con el token JWT para la autenticación.
  • Temporizadores: Simulan el tiempo de espera entre las acciones de los usuarios, para que la prueba sea más realista.
  • Controladores Lógicos: Permiten controlar el flujo de ejecución de la prueba, como la ejecución condicional de peticiones o la repetición de un grupo de peticiones.
  • Aserciones: Verifican que las respuestas del servidor cumplen con las expectativas, como la presencia de un código de estado HTTP 200 (OK) o la presencia de un elemento específico en la respuesta.
  • Listeners: Muestran los resultados de la prueba en diferentes formatos, como tablas, gráficos y árboles de resultados. Permiten visualizar las métricas de rendimiento, como los tiempos de respuesta, el número de peticiones por segundo, los errores y el consumo de recursos.

7. Pruebas y Validación

La calidad del software es una prioridad en "Arrienda tu Finca". Para garantizar la funcionalidad, el rendimiento y la seguridad de la aplicación, se ha implementado una estrategia de pruebas exhaustiva que incluye diferentes tipos de pruebas:

Pruebas Unitarias

Las pruebas unitarias se centran en verificar el correcto funcionamiento de las unidades individuales de código, como clases y métodos. Estas pruebas se realizan de forma aislada, utilizando mocks para simular las dependencias externas.

Herramientas utilizadas:

  • JUnit: Framework de pruebas unitarias para Java, que proporciona anotaciones y aserciones para escribir y ejecutar pruebas.
  • Mockito: Framework para la creación de mocks (objetos simulados) en Java, que permite simular el comportamiento de las dependencias externas y aislar el código bajo prueba.

Ejemplos de pruebas unitarias:

  • Verificar que el método listarFincas() del servicio FincaService devuelve la lista correcta de fincas de la base de datos.
  • Verificar que el método validarToken() del servicio JWTTokenService valida correctamente un token JWT.

Pruebas de Integración

Las pruebas de integración verifican la interacción entre diferentes componentes del sistema. Estas pruebas se realizan para asegurar que los componentes funcionan correctamente cuando se integran entre sí.

Ejemplos de pruebas de integración:

  • Verificar la comunicación entre el frontend y el backend, comprobando que las solicitudes HTTP se realizan correctamente y que las respuestas se procesan de forma adecuada.
  • Verificar la interacción con la base de datos, comprobando que los datos se guardan, recuperan y actualizan correctamente.
  • Verificar la integración entre diferentes servicios del backend, como la interacción entre el servicio de autenticación y el servicio de fincas.

Pruebas de Extremo a Extremo (E2E)

Las pruebas de extremo a extremo validan el flujo completo de la aplicación desde la perspectiva del usuario. Estas pruebas simulan el comportamiento de un usuario real interactuando con la aplicación, navegando por las diferentes pantallas, ingresando datos y verificando los resultados.

Herramienta utilizada:

  • Cypress: Herramienta de pruebas E2E para aplicaciones web, que permite simular la interacción del usuario en un navegador real.

Ejemplos de pruebas E2E:

  • Simular el proceso de registro de un nuevo usuario, incluyendo el ingreso de datos en el formulario de registro, la verificación del correo electrónico y el inicio de sesión en la aplicación.
  • Simular el proceso de publicación de una finca por parte de un arrendador, incluyendo el ingreso de la información de la finca, la subida de fotos y la verificación de que la finca se muestra correctamente en la lista de fincas disponibles.
  • Simular el proceso de búsqueda y solicitud de arrendamiento de una finca por parte de un arrendatario, incluyendo la búsqueda de fincas, la selección de una finca, el envío de la solicitud de arrendamiento y la verificación del estado de la solicitud.

Pruebas de Carga

Las pruebas de carga evalúan el rendimiento del sistema bajo estrés, simulando el acceso concurrente de múltiples usuarios a la aplicación.

Herramienta utilizada:

  • JMeter: Herramienta de pruebas de carga de código abierto, que permite simular la carga de usuarios concurrentes en una aplicación web.

Ejemplos de pruebas de carga:

  • Simular un gran número de usuarios iniciando sesión en la aplicación simultáneamente.
  • Simular múltiples usuarios buscando fincas con diferentes criterios de búsqueda.
  • Simular múltiples usuarios enviando solicitudes de arrendamiento a diferentes fincas.

La estrategia de pruebas implementada en "Arrienda tu Finca" aporta los siguientes beneficios:

8. Despliegue

Para asegurar que "Arrienda tu Finca" esté disponible de forma confiable y eficiente para los usuarios, se ha elegido una estrategia de despliegue basada en contenedores utilizando Docker y Kubernetes.

Docker

Docker es una plataforma de código abierto que permite empaquetar aplicaciones y sus dependencias en contenedores. Estos contenedores son unidades de software portátiles y autocontenidas que se pueden ejecutar en cualquier sistema que tenga Docker instalado, independientemente del sistema operativo subyacente.

Beneficios de usar Docker:

  • Portabilidad: Los contenedores Docker se pueden ejecutar en cualquier máquina con Docker instalado, lo que facilita el despliegue en diferentes entornos (desarrollo, pruebas, producción).
  • Consistencia: Los contenedores garantizan que la aplicación se ejecutará de la misma manera en cualquier entorno, eliminando problemas de compatibilidad.
  • Aislamiento: Los contenedores aíslan la aplicación y sus dependencias del sistema operativo anfitrión, lo que mejora la seguridad y la estabilidad.
  • Eficiencia: Los contenedores son ligeros y consumen menos recursos que las máquinas virtuales tradicionales.

En "Arrienda tu Finca", se utilizan contenedores Docker para empaquetar tanto el frontend (Angular) como el backend (Spring Boot) de la aplicación. Esto permite desplegar la aplicación de forma consistente en diferentes entornos.

Kubernetes

Kubernetes es un sistema de código abierto para la orquestación de contenedores. Permite automatizar el despliegue, la escalabilidad y la gestión de aplicaciones en contenedores.

Beneficios de usar Kubernetes:

  • Automatización del despliegue: Kubernetes automatiza el despliegue de aplicaciones en contenedores, lo que facilita la actualización y el lanzamiento de nuevas versiones.
  • Escalabilidad: Kubernetes permite escalar la aplicación horizontalmente, añadiendo o eliminando instancias de la aplicación según sea necesario para manejar la carga.
  • Alta disponibilidad: Kubernetes monitoriza el estado de la aplicación y reinicia automáticamente las instancias que fallen, lo que garantiza la disponibilidad del sistema.
  • Gestión de recursos: Kubernetes gestiona los recursos del clúster, asignando recursos a las aplicaciones según sea necesario.
  • Auto-recuperación: Kubernetes puede reiniciar automáticamente los contenedores que fallen, lo que aumenta la fiabilidad del sistema.
  • Descubrimiento de servicios: Kubernetes facilita el descubrimiento de servicios dentro del clúster, lo que permite que las aplicaciones se comuniquen entre sí de forma eficiente.

En "Arrienda tu Finca", Kubernetes se utiliza para desplegar y gestionar los contenedores Docker en un clúster de servidores. Esto permite escalar la aplicación de forma flexible, asegurar la alta disponibilidad y facilitar la gestión del sistema en producción.

Proceso de Despliegue

El proceso de despliegue de "Arrienda tu Finca" sigue los siguientes pasos:

  1. Construcción de imágenes Docker: Se crean imágenes Docker para el frontend y el backend de la aplicación, incluyendo todas las dependencias necesarias.
  2. Publicación de imágenes: Las imágenes Docker se publican en un registro de imágenes, como Docker Hub o un registro privado.
  3. Definición de los manifiestos de Kubernetes: Se crean archivos YAML que definen los recursos de Kubernetes, como deployments, services y pods, que se utilizarán para desplegar la aplicación.
  4. Despliegue en Kubernetes: Se utilizan las herramientas de Kubernetes (kubectl) para desplegar la aplicación en el clúster, utilizando los manifiestos definidos.
  5. Verificación del despliegue: Se verifica que la aplicación se ha desplegado correctamente y que está funcionando como se espera.

Beneficios del Despliegue con Docker y Kubernetes

El uso de Docker y Kubernetes para el despliegue de "Arrienda tu Finca" aporta los siguientes beneficios:

  • Despliegue rápido y eficiente: La automatización del despliegue con Docker y Kubernetes permite realizar despliegues rápidos y eficientes, lo que reduce el tiempo de inactividad de la aplicación.
  • Escalabilidad y alta disponibilidad: Kubernetes permite escalar la aplicación de forma flexible y asegurar la alta disponibilidad del sistema, lo que garantiza que la aplicación pueda manejar la carga de usuarios y que esté siempre disponible.
  • Gestión simplificada: Kubernetes facilita la gestión y el mantenimiento de la aplicación en producción, proporcionando herramientas para monitorizar el estado del sistema, actualizar la aplicación y solucionar problemas.
  • Portabilidad: La aplicación se puede desplegar en diferentes entornos (nube, on-premise) sin necesidad de realizar cambios significativos en la configuración.
